## Поток событий для прецедента «Просмотреть все сохраненные материалы»
### Главный поток
1. Вариант использоывания начинается, когда пользователь выбирает в меню приложения пункт "My List"
2. Приложение выводит список всех сохраненных материалов. Если сохраненных маериалов нет, выполняется альтернативный поток A1.
3. Вариант использования завершается.
### Альтернитивный поток A1
1. Поток начинается.
2. Пользователю выводится сообщение, что у него нет сохраненных материалов.
3. Вариант использования завершается.
## Поток событий для прецедента «Сохранить материал»
### Главный поток
1. Вариант использования начинается, когда пользователь нажимает на кнопку "+".
2. Приложение отображает модальное окно с полями для ввода информации
3. Пользователь вводит название материала. Если пользователь ничего не ввел в поле для названия материала, выполняется альтернативный поток Е1.
4. Пользователь перетаскивает картинку в поле drag&drop для прикрепления ее к материалу.Если картинка не прикрепилась, выполняется альтернативный поток Е1.
5. Пользователь вводит теги. Если пользователь ничего не ввел в поле для тегов, выполняется альтернативный поток Е1.
6. Пользователь вводит ссылку на ресурс. Если пользователь ничего не ввел в поле для ссылки, выполняется альтернативный поток Е1.
7. Пользователь нажимает на кнопку "Add". Если пользователь кликнет вне модального окна - выполняется альтернативный поток A1.
8. Материаль добавляется в список сохраненных материалов.
9. Вариант использования завершается.
### Альтернативный поток А1
1. Поток начинается.
2. Модальное окно закрывается.
3. Вариант использования завершается.
### Альтернативный поток Е1
1. Поток начинается
2. Фон поля для ввода, в котором была допущена ошибка, изменяется на красный. Под полем пользователю отображается сообщение об ошибке.
3. Поток завершается.
## Поток событий для прецедента «Добавить материал в избранное»
### Главный поток
1. Вариант использования начинается, когда пользователь наводит курсором на форму материала.
2. В форме материала появляется иконка "Favorite" в виде звезды.
3. Пользователь кликает по иконке. Если материал уже добавлен в избранное, выполняется альтернативный поток А1.
4. Иконка меняет цвет фона.
5. Выбранный материал добавляется в список избранных материалов.
6. Вариант использования завершается.
### Альтернативный поток А1
1. Поток начинается.
2. Иконка меняет цвет фона.
3. Выбранный материал удаляется из списка избранного.
4. Вариант использования завершается.
## Поток событий для прецедента «Просмотреть материалы, добавленные в избранное»
### Главный поток
1. Вариант использования начинается, когда пользователь выбирает в меню приложения пункт "Favorite".
2. Приложение отображает список материалов, добавленных в избранное. Если материалов, добавленных в избранное, нет, выполняется альтернативный поток А1.
3. Вариант использования завершается.
### Альтернативный поток А1
1. Поток начинается
2. Приложение выводит сообщение, что у пользователя нет материалов, добавленных в избранное.
3. Поток завершается.
 ## Поток событий для прецедента «Просмотреть теги»
### Главный поток
1. Вариант использования начинается, когда пользователь выбирает в меню приложения пункт "Tags".
2. Приложение отображает все теги, существующие в приложении, в виде облака тегов.
3. Вариант использования завершается.
 ## Поток событий для прецедента «Удалить материал»
### Главный поток
1. Вариант использования начинается, когда пользователь наводит курсор на форму сохраненного материала.
2. Приложение отображает иконку "Trash" в виде мусорного бака.
3. Пользователь кликает на иконку "Trash".
4. Приложение удаляет выбранный материал из списка мохраненных материалов.
5. Вариант использования завершается.
 ## Поток событий для прецедента «Поиск материала по тегу»
### Главный поток
1. Вариант использования начинается, когда пользователь кликает на тег из облака тегов.
2. Приложение открывает пункт "My List", где выводит все материалы с заданным тегом.
3. Вариант использования завершается.
## Поток событий для прецедента «Перейти к прочтению материала
»
